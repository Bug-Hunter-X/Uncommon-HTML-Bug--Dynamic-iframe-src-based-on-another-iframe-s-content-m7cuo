This is an example of an uncommon HTML bug that can occur when using nested iframes. The problem arises when an iframe's src attribute is dynamically set based on the content of another iframe. The solution uses a combination of event listeners and asynchronous loading to ensure the data is ready before trying to access it.

<iframe id="iframe1" src="iframe1.html"></iframe>
<iframe id="iframe2"></iframe>
<script>
  const iframe1 = document.getElementById('iframe1');
  iframe1.onload = () => {
    const iframe1Content = iframe1.contentWindow.document.body.textContent;
    const iframe2 = document.getElementById('iframe2');
    iframe2.src = iframe1Content.trim() === 'valid' ? 'iframe2valid.html' : 'iframe2invalid.html';
  };
</script>
